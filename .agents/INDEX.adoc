== Notes Session 005

- *Blocage Google contourné* : Chromium snap lancé en subprocess normal (pas via Playwright
  LaunchPersistentContext qui ajoute --enable-automation détecté par Google). Playwright
  se connecte ensuite via connectOverCDP("http://localhost:9222"). Google voit un navigateur
  normal, pas un navigateur automatisé.
- *Flags anti-détection* : --disable-blink-features=AutomationControlled, --no-first-run,
  --no-default-browser-check. Profil Chromium snap Profile 2 (authentifié Google Colab)
  via --user-data-dir + --profile-directory.
- *Run all* : raccourci clavier Ctrl+F9 (bypass shadow DOM Polymer de Colab qui rend les
  selectors UI fragiles).
- *Polling git* : remplace polling DOM (colab-terminal, .output_text, pre) qui ne trouvait
  jamais TRANSLATION_DONE=ok dans le shadow DOM. Maintenant : git ls-remote sur
  cheroliv/office.git, détecte changement de HEAD sha → succès. Timeout 30 min.
- *Architecture plugin indépendant* : conformément à l'article 0124, le sous-module
  container-provision-plugin/ est désormais un build Gradle autonome avec son propre
  gradlew + settings.gradle.kts. La racine ne fait plus include() — elle consomme le
  plugin via mavenLocal. Ajout du plugin maven-publish pour exposer publishToMavenLocal.
- *RESTE CP-1d* : Le notebook Colab ne push pas après 30 min. Le pipeline Gradle→Playwright→
  Chromium→Colab est validé (Ctrl+F9 envoyé, exécution démarrée). Le blocage est côté
  notebook lui-même : secrets Colab (github-api-key-cheroliv, REPO_OFFICE) à vérifier,
  ou erreur dans une cellule (Ollama pull translategemma:4b, clone office.git, etc.).
