== Décisions techniques

1. *BOM comme registre central* — Le toml est la source unique de vérité pour
   les versions des boroughs. Tous les consommateurs (Colab, cross-boroughs)
   consomment `platform("education.cccp:workspace-bom:0.0.5")`.
2. *buildscript + apply* — Les `plugins {}` blocks de Gradle ne consomment pas
   les BOM. Pour que le BOM contraigne le classpath plugin (et donc annotations),
   le consommateur doit utiliser `buildscript { classpath(platform(...)) }` +
   `apply(plugin=...)` + `extensions.configure<T>("name")`.
3. *Publication avant test Colab* — Le BOM doit être sur Maven Central pour
   que Colab (qui n'a pas mavenLocal) puisse le résoudre.
